# Outputs

Publication-ready deliverables for the Pricing Discount Governance System. Everything here is generated from the processed data by reproducible scripts.

```
outputs/
├── reports/     Analytical PDF report (start here)
├── graphs/      The chart pack used throughout the report
├── dashboard/   Standalone interactive command center (open the .html directly)
├── warehouse/   SQL model logs, manifest, and validation report
└── release/     Release gate reports from the latest pipeline run
```

## Reports

[`pricing_discount_governance_report.pdf`](reports/pricing_discount_governance_report.pdf) is a 34-page diagnostic of pricing health across the $1.88B synthetic revenue book (January 2023 to December 2025). It includes an executive summary, methodology, findings, sensitivity analysis, caveats, and prioritized recommendations.

The headline: revenue is stable while discount depth remains elevated. Price realization sits at 81.8%, meaning $419M of modeled list value is forgone to discount over the period, with the highest-priority exposure concentrated in Enterprise accounts.

Rebuild with `python scripts/build_report_pdf.py` (after the pipeline and `scripts/build_report_assets.py` have run).

## Graphs

Eighteen charts cover trend, distribution, composition, correlation, ranking, concentration, geography, risk, cohort, and threshold sensitivity. They use a consistent editorial palette and direct labeling. Generated by `scripts/build_report_assets.py`.

## Dashboard

[`pricing-discipline-command-center.html`](dashboard/pricing-discipline-command-center.html) opens locally with no server or build step. Data is embedded inline and Chart.js ships in `vendor/`; optional Google Fonts fall back to system fonts when offline. It carries KPI cards, scope filters, and four interactive charts that re-slice the book by segment, region, category, and channel.

Published version: https://mfidalgomartins.github.io/pricing-discount-governance-system/

## Operational Files

Generated diagnostics in the root of `outputs/` are designed for audit and troubleshooting:

| File | Purpose |
|---|---|
| `run_manifest.json` | schema version, run parameters, runtime versions, row counts, validation state, publication state, and duration |
| `raw_validation_report.csv` | source contract checks |
| `processed_validation_report.csv` | processed table checks |
| `metric_contract_validation.csv` | published metric contract checks |
| `final_validation_summary.json` | final readiness state and blocker counts |
| `final_validation_review.md` | human-readable validation review |
| `release/release_gate_report.md` | release policy pass/fail summary |

Most CSV/JSON/Markdown diagnostics are ignored by git and should be regenerated rather than hand-edited.

## Rebuild Order

```bash
make run
make report
python scripts/release_gate.py
make preflight
```

Do not publish if the release gate fails or if validation reports contain unresolved `FAIL` rows.

All figures trace to the pipeline's processed marts. Ten formal-analysis validation checks and the full metric-contract reconciliation passed at publication time. The underlying data is synthetic; the method is designed to be adapted and revalidated against a real order book.
