# Outputs

Publication-ready deliverables for the Pricing Discount Governance System. Everything here is generated from the processed data by reproducible scripts.

```
outputs/
├── reports/    Analytical PDF report (start here)
├── graphs/     The chart pack used throughout the report
└── dashboard/  Standalone interactive command center (open the .html directly)
```

## Reports

[`pricing_discount_governance_report.pdf`](reports/pricing_discount_governance_report.pdf) is a 33-page diagnostic of pricing health across the $1.87B synthetic revenue book (January 2023 to December 2025). It includes an executive summary, methodology, findings, sensitivity analysis, caveats, and prioritized recommendations.

The headline: revenue is stable but bought with a structurally elevated discount. Price realization sits at 81.8%, meaning $417M of list value is forgone to discount over the period, and the exposure concentrates in 35 large Enterprise accounts.

Rebuild with `python scripts/build_report_pdf.py` (after the pipeline and `scripts/build_report_assets.py` have run).

## Graphs

Eighteen charts, one per analytical question, covering trend, distribution, composition, correlation, ranking, concentration, geography, risk, cohort, and threshold sensitivity. Consistent editorial palette with a single accent color, direct labeling, and no chart junk. Generated by `scripts/build_report_assets.py`.

## Dashboard

[`pricing-discipline-command-center.html`](dashboard/pricing-discipline-command-center.html) opens locally with no server or build step. Data is embedded inline and the only local dependency, Chart.js, ships in `vendor/`. It carries KPI cards, scope filters, and four interactive charts that re-slice the book by segment, region, category, and channel.

Published version: https://mfidalgomartins.github.io/pricing-discount-governance-system/

---

All figures trace to the pipeline's processed marts. Ten formal-analysis validation checks and the full metric-contract reconciliation passed at publication time. The underlying data is synthetic; the method is designed to be adapted and revalidated against a real order book.
